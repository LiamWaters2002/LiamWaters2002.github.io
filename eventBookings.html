Project Overview

The event booking website was university coursework for Internet Applications and Database Design. This was a first-year module in which I had to build a website, something I had never done before.
I achieved a 96% in that module, in which the website contributed to 80% of the final mark, 20% were from weekly assessments (9 in total).

Technologies Used

  • HTML
  • CSS
  • JavaScript
  • PHP
  • MySQL

Key Features

  • Filter System (categories)
  • Order System (Date, time)
  • Dynamic Content from Relational Database
  • Like and Booking System
  • Register and Login System
  • Responsive UI design

What's next?

Since this was my first-ever project, there are things I would like to improve and add that I learned during my years learning computer science.

  1. UI design: Improve the UI to make it less 'boxy' and reduce the amount of color as it is distracting. Some UI design choices I made when making the website in the first year were responsive design.

  2. Payment System: Utilize Shopify API for events that require payment. This was not a requirement for the assessment, but it is something I plan to add.

  3. Event Organiser Role: Add the ability to create new events.

  4. Admin Role: Implement the ability to grant event organizer roles to specific accounts.

  5. Filter to only display liked/booked events only.
